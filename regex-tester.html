Flags reference

FlagNameWhat it does
gGlobalFind all matches instead of stopping at the first one. Always applied here so you can see every match; remember to add it in your own code.
iIgnore caseMatch letters regardless of case — "ABC" matches "abc".
mMultilineMakes ^ and $ match the start/end of each line, not just the whole string.
sDot-allMakes . match newline characters too, which it normally doesn't.
uUnicodeTreats the pattern as a sequence of Unicode code points, important for patterns matching emoji or non-Latin scripts.

Frequently asked questions

Why does the match list show all matches even without the g flag checked?

For testing purposes, DevLabs always searches for every match in your test string so you can see the full picture. In your actual code, you still need the g flag if you want String.match or a matchAll loop to return more than the first result.

What happens if my pattern is too slow?

Patterns with nested quantifiers, like (a+)+, can cause catastrophic backtracking — where matching time grows exponentially with input length — which would otherwise freeze the page solid. DevLabs normally runs matching in a background worker with a timeout, so a runaway pattern is stopped automatically instead of locking up your browser tab. If your browser blocks that background worker, the tool switches to a safer fallback: live-as-you-type turns off, and you'll get a warning before running any pattern that looks like it could be slow.

Does this use JavaScript regex syntax specifically?

Yes. Regex flavors differ slightly between languages (PCRE, Python, .NET, etc.). This tool uses your browser's native JavaScript regex engine, so results will match what you'd get from a JS RegExp exactly, but may differ slightly from PCRE-based tools for advanced features like lookbehind support in older engines.
